Microsoft se rinde ante GitHub y cierra CodePlex

Después de 11 años de actividad, Microsoft anuncia el cierre de CodePlex, su sitio web de repositorios de código que competía con GitHub y SourceForge

Sabiendo que GitHub nació en 2008, es obvio que en su día CodePlex nació para competir con el aún más veterano SourceForce, frente al cual intentaba ofrecer mejores funciones y un menor bombardeo de publicidad (unido a una mala gestión que puso SourceForce contra las cuerdas). Sin embargo, el nacimiento de GitHub y su alzamiento como la gran referencia de los repositorios de código relegó a CodePlex a la irrelevancia. Ante esa situación, la misma Microsoft terminó reconociendo indirectamente su propia derrota al publicar en GitHub el código de todos sus proyectos Open Source relevantes, incluidos Visual Studio Code y .NET.

El sitio web de repositorios de código de Microsoft no es el primero en caer ante el dominio de GitHub, ya que en 2015 se anunció el cierre de Google Code por circunstancias parecidas.

Esto no significa que CodePlex vaya a ser cerrado bruscamente, sino que los repositorios existentes podrán seguir actualizándose hasta octubre de 2017. A partir de ahí todos los repositorios pasarán a ser de solo lectura hasta el 15 de diciembre de 2017, cuando el sitio web será cerrado definitivamente.

Si tienes un proyecto alojado en CodePlex, lo recomendable sería migrarlo a otro sitio web de repositorios cuanto antes.
